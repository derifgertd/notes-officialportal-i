# notes-officialportal-i

本仓库用于归档和发布多个独立 HTML 页面。这些页面以静态文件形式存放，主要服务于信息记录与展示目的。

## 项目简介

该仓库集中管理一批独立的 HTML 页面，每个页面均为完整的自包含文件，不依赖外部资源或特定后端服务。页面内容可能涉及不同主题，但均以离线可查看的 HTML 格式存储，便于分发、存档或通过 GitHub Pages 等静态托管服务进行展示。

## 目录说明

```
.
├── README.md          # 本文件
├── index.html         # 入口页面（如有）
├── pages/             # 存放各独立 HTML 页面
│   ├── page-1.html
│   ├── page-2.html
│   └── ...
└── assets/            # 页面共享资源（如图片、样式、脚本等）
```

- `pages/`：核心目录，存放所有独立 HTML 页面，文件名按内容或编号命名。
- `assets/`：可选目录，存放页面所需的共享静态资源，如样式表、图片或 JavaScript 文件。各页面也可选择内嵌资源以保持独立性。

## 页面归档说明

- 每个 HTML 文件均为独立页面，可直接在浏览器中打开使用。
- 页面之间无强制关联，可单独访问或通过入口页面导航。
- 页面内容以静态文本、表格、列表等常见 HTML 元素为主，不包含动态数据请求或第三方服务调用。
- 所有页面均经过基础校验，确保结构完整、编码正确（UTF-8）。

## 维护说明

- 新增页面：在 `pages/` 目录下创建新的 HTML 文件，命名遵循已有规范（如按时间或序号），并确保页面自包含。
- 更新页面：直接修改对应 HTML 文件，修改后建议进行本地预览检查。
- 删除页面：从 `pages/` 目录中移除文件，并更新入口页面（如有）中的链接。
- 本仓库不收集用户信息，不包含跟踪代码，仅作为静态页面存档使用。

## 其他

- 本仓库内容仅用于归档展示，不构成任何形式的服务承诺或运营保证。
- 如通过 GitHub Pages 部署，请确保仓库设置中已启用 Pages 服务，并选择正确的发布源分支及目录。
